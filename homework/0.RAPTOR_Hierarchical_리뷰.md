# RAPTOR: 계층형 RAG (Hierarchical RAG) 개요 및 비교 분석

최근 RAG(검색 증강 생성) 기술의 고도화와 함께 등장한 **RAPTOR(Recursive Abstractive Processing for Tree-Organized Retrieval)** 기법의 등장 배경과 기존 RAG 기법들과의 차이점을 정리합니다.

---

## 1. 등장 배경: "나무만 보고 숲은 보지 못한다"

기존의 기본형 RAG 모델들은 방대한 문서를 일정 크기의 텍스트 조각(Chunk)으로 잘게 쪼갠 뒤, 사용자의 질문과 가장 유사한 조각을 찾아 답변을 생성합니다. 

이 방식은 세부적인 팩트를 찾는 데는 탁월하지만, **전체적인 문맥 파악(Holistic understanding)이 불가능하다는 치명적인 단점**이 있었습니다. (예: "해리포터 전권을 관통하는 핵심 주제는?"이라는 질문에 지엽적인 텍스트만 가져오는 문제 발생)

RAPTOR는 이러한 **단편적 검색(Myopic Retrieval)의 한계를 극복하고, 세부 정보와 전체 문맥을 동시에 이해할 수 있는 시스템**을 만들기 위해 고안되었습니다.

## 2. 작동 원리: 재귀적 요약을 통한 트리 구조 구축

RAPTOR는 문서를 단순히 쪼개는 것에서 끝나지 않고, 쪼개진 텍스트들을 **'클러스터링(군집화)'**하고 **'요약(Abstractive Processing)'**하여 위로 쌓아 올리는 트리(Tree) 구조를 만듭니다.

1. **상향식(Bottom-up) 트리 구축:** * 문서를 기본 청크로 나눔 (잎 노드)
   * 내용이 비슷한 청크들을 묶어 LLM을 통해 요약본 생성 (부모 노드)
   * 요약본들을 다시 묶어 더 큰 요약본 생성 (조부모 노드)
   * 최상위 요약본이 나올 때까지 반복
2. **다층적 검색:** * 질문 입력 시, 잎사귀(세부 원문)부터 뿌리(전체 요약문)까지 트리의 모든 계층에서 질문과 관련된 정보를 동시에 검색

## 3. 기존 대표 RAG 기법과의 비교

가장 널리 쓰이는 두 가지 기존 RAG 기법과 RAPTOR의 차별점은 다음과 같습니다.

### ① Naive RAG (단순 청킹 기반 RAG) vs. RAPTOR
* **Naive RAG의 한계:** 문서를 1차원적으로 잘라서 저장. 문장 간 연결 고리나 전체 구조를 잃어버려 다중 홉(Multi-hop) 질문에 매우 취약함.
* **RAPTOR의 해결:** 상위 노드(요약본)가 여러 청크를 종합해 둔 결과물이므로, 넓은 시야가 필요한 질문(숲)과 세부 팩트(나무)를 모두 검색 가능.

### ② Parent Document Retriever (부모-자식 문서 검색 RAG) vs. RAPTOR
* **기존 기법의 한계:** 큰 '부모 문서'와 잘게 쪼갠 '자식 문서'로 나누어 검색은 자식으로, 문맥은 부모로 제공. 하지만 부모 문서가 단순히 '원본 텍스트의 큰 덩어리'일 뿐이라 LLM의 컨텍스트 창을 초과하거나 노이즈 텍스트까지 처리하는 비용/성능 문제 발생.
* **RAPTOR의 차별점:** 상위 노드로 갈 때 원문을 단순히 합치는 것이 아니라, **LLM이 능동적으로 "새로운 요약문(Abstractive Summary)"을 생성**하여 저장. 정보 밀도를 높여 노이즈 없이 고차원적인 문맥 정보 제공.

---

## 4. 요약: RAPTOR가 해결한 핵심 과제

| 비교 항목 | Naive RAG | Parent Document Retriever | RAPTOR |
| :--- | :--- | :--- | :--- |
| **구조** | 1차원 (평면적 청크) | 2단계 계층 (원본 자르기) | 다단계 트리 (재귀적 요약) |
| **상위 데이터 형태** | 없음 | 길이가 긴 **원본 텍스트** 덩어리 | LLM이 압축한 **요약된 텍스트** |
| **포괄적 질문 처리** | 매우 낮음 | 보통 (컨텍스트 길이에 의존) | **매우 높음** |
| **노이즈 처리 효율** | 낮음 (엉뚱한 청크 검색 시) | 낮음 (부모 원문을 통째로 제공) | **높음** (핵심만 요약되어 있음) |

**결론:** RAPTOR는 "방대한 문서에서 흩어진 정보를 어떻게 인간처럼 맥락을 파악하며 종합할 수 있을까?"라는 과제를 **재귀적 요약(Recursive Summarization)**이라는 기법으로 해결한 최신 아키텍처입니다.
